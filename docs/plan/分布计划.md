# 认证体系与备份同步迁移方案

## 摘要

PixezServer 按单租户部署设计迁移，不引入 `owner_user_id`、租户隔离或管理员跨用户规则。迁移顺序是：先注册 PixEz API 空壳并统一返回待开发；再把 Flutter custom 同步认证从 Basic Auth 改为 Wavelet AccessToken；最后实现 Pixiv 账号管理与 7 张备份同步表。镜像、收藏导出和文件能力不在本阶段实现。

## 阶段 1：API 空壳

- 在 `PixezServer/internal/apps/pixez/` 新增 PixEz 模块，只注册本阶段接口：
    - `GET /api/pixez/ping`
    - `GET /api/pixez/users`
    - `GET /api/pixez/users/:pixiv_user_id`
    - `PUT /api/pixez/users/:pixiv_user_id`
    - `DELETE /api/pixez/users/:pixiv_user_id`
    - `GET /api/pixez/users/:pixiv_user_id/sync-data`
    - `POST /api/pixez/users/:pixiv_user_id/sync-data`
    - `GET /api/pixez/users/:pixiv_user_id/sync-data/hashes`
- 路由只在 `PixezServer/internal/router/router.go` 注册，挂载 `oauth.LoginRequired()`。
- 所有空壳接口先返回：
  ```json
  { "error_msg": "待开发", "data": null }
  ```
- 暂不注册镜像、收藏导出、镜像管理接口，避免误用未迁移能力。

## 阶段 2：认证体系迁移

- 后端彻底废弃 PixEz Basic Auth，不再设计或读取 `PIXEZ_AUTH_USER` / `PIXEZ_AUTH_PASS`。
- Flutter custom 配置直接移除 `username/password`：
    - 删除 `SyncConfig.username` / `SyncConfig.password` 及其持久化 key。
    - 同步设置页移除用户名、密码输入框。
    - `SyncApi.getDioClient()` 只注入 AccessToken header。
- 新增或保留的客户端配置只包含：
    - `serverUrl`
    - `accessToken`
    - `enabled`
    - `syncInterval`
    - 其他仍在用的业务开关，例如 `autoMirrorNovels`，但镜像阶段再启用相关 UI。
- 客户端认证头统一使用 Wavelet 支持的形式：
  ```http
  Authorization: Bearer <access_token>
  ```
- `ping()`、账号同步、备份同步响应判断统一改为：
    - HTTP 200 且 `error_msg` 为空字符串表示成功。
    - 失败提示从 `error_msg` 读取。
- AccessToken 获取路径不在 Flutter 内重做登录流程：用户在 PixezServer Web 管理端登录后，从现有 AccessToken 管理功能创建 token，再填入 Flutter 同步设置页。

## 阶段 3：备份同步实现

- 新增 goose 双方言迁移，创建旧 server 同语义表：
    - `pixiv_users`
    - `ban_comments`
    - `ban_illusts`
    - `ban_tags`
    - `ban_users`
    - `illust_histories`
    - `novel_histories`
    - `tag_histories`
- 不新增 `owner_user_id`。`pixiv_user_id` 继续作为 Pixiv 账号和备份数据的归属边界。
- 表字段保持旧客户端 payload 兼容；服务端写入时忽略客户端传入的隐藏归属字段，统一使用 path 中的 `pixiv_user_id`。
- `PUT /api/pixez/users/:pixiv_user_id`：
    - path `pixiv_user_id` 为权威值。
    - 保存 Pixiv 用户资料、access token、refresh token、device token。
    - 返回 `{ "error_msg": "", "data": null }`。
- `GET /api/pixez/users`：
    - 返回全部已保存 Pixiv 用户 safe DTO。
    - 不返回 token 字段。
- `GET /api/pixez/users/:pixiv_user_id`：
    - 返回完整 Pixiv 凭据，用于客户端一键恢复登录。
- `DELETE /api/pixez/users/:pixiv_user_id`：
    - 删除该 Pixiv 用户及 7 张备份表中对应数据。
    - 本阶段不处理镜像、收藏导出等未迁移表。
- `POST /api/pixez/users/:pixiv_user_id/sync-data`：
    - 事务内替换请求中提交的表。
    - 未提交表不修改，支持选择性同步。
    - 每条记录强制写入 path 中的 `pixiv_user_id`。
- `GET /api/pixez/users/:pixiv_user_id/sync-data`：
    - 支持 `tables=ban_comments,illust_histories` 选择性下载。
    - 返回字段保持 Flutter custom 现有模型兼容。
- `GET /api/pixez/users/:pixiv_user_id/sync-data/hashes`：
    - 按旧 server 规则排序后计算 MD5。
    - 空表返回 `"empty"`。
    - hash 不包含本地自增 ID 和时间戳。

## Flutter 侧改造

- 修改 `lib/custom/services/sync_config.dart`：删除 username/password 配置与持久化。
- 修改 `lib/custom/pages/sync_settings_page.dart` 和 `sync_login_page.dart`：移除 Basic Auth 输入，新增 AccessToken 输入。
- 修改 `lib/custom/services/sync_api.dart`：
    - 删除 `_basicAuth()`。
    - 统一通过 `_authHeaders()` 写入 Bearer token。
    - 新增统一解包函数，兼容 Wavelet `{error_msg,data}`。
    - 所有旧 `success == true` 判断改为 `error_msg == ""`。
- 保持 custom 层集中改造，不触碰原 PixEz 网络层。

## 测试与验收

- 后端测试：
    - 无 AccessToken 访问 `/api/pixez/**` 返回 401。
    - 有效 AccessToken 可访问空壳和真实同步接口。
    - 用户列表不返回 token。
    - `PUT/GET/DELETE pixiv_users` 行为正确。
    - `POST sync-data` 只替换提交表，未提交表保留。
    - `GET sync-data?tables=...` 只返回指定表。
    - hash 与旧 server 同 payload 结果一致。
- Flutter 验证：
    - 同步设置页不再出现用户名/密码。
    - AccessToken 保存后 `ping` 可通过。
    - 账号上传、账号下载、备份上传、选择性恢复仍可用。
- 门禁：
    - PixezServer 跑相关 Go tests。
    - API 注释变更后跑 `make swagger`。
    - 提交前跑 `make code-check`。
    - Flutter custom 层改动后跑 `flutter analyze`。

## 假设

- PixezServer 是单租户伴生服务，一个服务实例服务一个可信同步域。
- 本阶段不做 Wavelet 移动端登录页，只复用现有 Web 管理端 AccessToken。
- 不保留 Basic Auth 兼容配置；不用的 username/password 配置直接删除。
