# PixEz 后端迁移到 PixezServer/Wavelet 计划

## 摘要

将旧 `server/` 的 PixEz Sync 业务迁移进 `PixezServer/`，不平移旧框架。新实现采用 Wavelet 的 Gin 路由、GORM、goose 双方言迁移、AccessToken 登录体系、Asynq 异步任务、TaskExecution 可观测性和 Upload/File 存储能力。

兼容定义锁定为：保留现有 PixEz 业务路径和主要请求/响应 payload 语义，例如 `/api/pixez/**` 与 `/mirror/**`，但统一响应包裹改为 Wavelet 标准 `{ "error_msg": "", "data": ... }`，认证改为 Wavelet `Authorization: Bearer <access_token>` 或 `X-Access-Token`。

## 当前进度与本阶段边界

- 已完成：`PixezServer/` 已接入 `/api/pixez/ping`、Pixiv 用户 CRUD、7 张同步数据表、`sync-data` 上传/下载/hash 接口，Flutter custom 同步认证已从 Basic Auth 改为 Wavelet AccessToken。
- 本阶段继续补齐：镜像 read-model、收藏导出 read-model、`/mirror/**` 读取、PixEz Asynq 任务、旧 `server/pixez-sync.db` 一次性导入和中文文档同步。
- 本阶段不恢复旧 `server/` 的 `mirror_tasks` 轮询队列；旧 `mirror_tasks` 只作为导入历史状态来源，新的队列、日志、重试和执行历史统一交给 Wavelet `task_executions`。
- 当前仓库没有 `docs/config.ts`，因此文档注册仅更新现有 `docs/design/index.md`、`docs/design/architecture.md`、`docs/reference/configuration.md`、`docs/reference/database.md` 和本计划索引。

## 关键改造

- 在 PixezServer 新增 `internal/apps/pixez/` 业务模块，承载 Pixiv 账号、同步数据、收藏导出、镜像管理和 Pixiv 形态 mirror 读取接口；所有路由只在 `internal/router/router.go` 注册。
- 新增 PixEz 业务模型与 goose 迁移，覆盖 `pixiv_users`、7 张同步数据表、`bookmark_export_runs`、`bookmark_illusts`、`bookmark_novels`、`mirror_illust`、`mirror_novel`；不继续把旧 `mirror_tasks` 当队列表使用。
- 镜像与收藏导出改为 Wavelet Asynq 任务：
  - `pixez:mirror_illust`
  - `pixez:mirror_novel`
  - `pixez:export_bookmark_illusts`
  - `pixez:export_bookmark_novels`
  - 可选 `pixez:auto_enqueue_bookmark_mirrors`
- 镜像状态由 PixEz read-model 表保存，任务执行历史、日志、重试、失败原因由 Wavelet `task_executions` 管理；业务状态接口返回 `task_id/status/mirrored/total_count/success_count/failed_count/retry_urls_json` 等客户端仍需要的字段。
- 图片原文件不再直接写 `{MirrorDir}/{illust_id}/`，改为通过 Wavelet 上传/存储体系登记 `uploads` 记录，`mirror_illust.image_files_json` 保存 Pixiv URL、page、Upload ID、文件名、hash、mime、size、storage key 映射；`/mirror/pximg/*path` 根据该映射流式读取文件。
- 小说镜像继续保存 Pixiv 原始详情 JSON 与正文 JSON，`/mirror/v1/novel/detail` 和 `/mirror/webview/v2/novel` 返回 Pixiv 形态原始响应，不套 `{error_msg,data}`。
- Pixiv 请求逻辑抽到 `internal/service/pixez`，统一处理 App API headers、User-Agent、refresh token、错误日志、placeholder 检测：插画 `limit_unknown_360`、小说 `limit_unknown_100`。
- 收藏导出保持已定语义：增量插入、跳过未变化 active 记录、placeholder 立即标记 removed、分页完整成功后才标记本轮缺失记录，禁止全量删改刷新。
- Flutter custom 层需要同步改造 `lib/custom/services/sync_api.dart`：认证从 Basic Auth 改为 AccessToken header，响应判断从 `success == true` 改为 `error_msg == ""`，`data` 解包逻辑保持集中在服务层，避免扩散到页面。
- 中文文档同步更新：新增/更新 PixezServer 设计文档、迁移计划、配置说明；旧 PixEz Sync backend 文档标注为“legacy server 设计”，避免未来误读。

## API/接口变更

- 保留业务路径：
  - `/api/pixez/ping`
  - `/api/pixez/users`
  - `/api/pixez/users/:pixiv_user_id`
  - `/api/pixez/users/:pixiv_user_id/sync-data`
  - `/api/pixez/users/:pixiv_user_id/sync-data/hashes`
  - `/api/pixez/users/:pixiv_user_id/bookmarks/illust/removed`
  - `/api/pixez/illusts/:illust_id/mirror`
  - `/api/pixez/illusts/mirror/batch`
  - `/api/pixez/novels/:novel_id/mirror`
  - `/api/pixez/novels/mirror/batch`
  - `/api/pixez/mirror/illusts`
  - `/api/pixez/mirror/novels`
- 所有 `/api/pixez/**` 统一返回 Wavelet envelope：
  ```json
  { "error_msg": "", "data": {} }
  ```
- `/mirror/**` 保持 Pixiv 官方响应形态或二进制文件输出，不使用系统 envelope。
- 认证统一使用 Wavelet 登录态：客户端使用 AccessToken，不再配置 `PIXEZ_AUTH_USER` / `PIXEZ_AUTH_PASS`。
- 后台管理可通过 Wavelet Admin 任务页查看 PixEz 任务类型、下发任务、重试失败任务和查看执行日志。

## 数据迁移

- 提供一次性迁移工具或 Admin 任务，从旧 SQLite `server/pixez-sync.db` 导入 PixEz 业务表到 PixezServer 数据库。
- 旧镜像文件迁移策略：
  - 读取旧 `mirror_illust.image_files_json` 与 `server/data/mirror/**`。
  - 对存在的原图文件计算 hash/mime/size，写入 Wavelet upload 存储与 `uploads` 表。
  - 更新新 `mirror_illust.image_files_json` 为 Upload ID 映射。
  - 缺失文件只保留元数据与失败 URL，不伪造成功文件。
- 导入过程必须可重复运行：以 Pixiv user ID、target type/id、bookmark unique key、upload hash 作为幂等依据。
- 旧 `mirror_tasks` 只作为历史状态参考导入，不作为新队列源；新任务由 Asynq 重新承担。

## 测试计划

- Go 单测：模型迁移、Pixiv payload 解析、placeholder 检测、sync-data hash、账号 CRUD、收藏增量/removed 语义、mirror status 语义。
- 任务测试：四类 PixEz Asynq handler 的 payload 校验、成功路径、失败路径、AppendLog、重试可见性。
- API 测试：AccessToken 鉴权、`/api/pixez/**` envelope、`/mirror/**` 非 envelope、批量 mirror status、删除镜像记录。
- 文件测试：旧文件导入、Upload 存储映射、`/mirror/pximg/*path` master/square 到 original 映射、文件缺失返回 404。
- 客户端验证：Flutter custom 层登录配置、ping、账号同步、备份恢复、插画 fallback、小说 mirror-first、镜像管理列表。
- 门禁：PixezServer 跑 `go test` 相关包、`make swagger`、`make code-check`；Flutter 侧跑 `flutter analyze`。

## 假设

- “保持客户端兼容”指保留业务路径和 payload 语义，不保留 Basic Auth 与 `{success,message,data}`。
- PixezServer 作为长期后端，旧 `server/` 迁移完成后只作为 legacy 参考，不继续双写。
- 文件统一走 Wavelet Upload/Storage；本地目录缓存只作为存储驱动实现细节，不暴露为 PixEz 独立 `MirrorDir` 配置。
- 只同步中文文档，不维护英文文档。
